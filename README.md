# 永远的山根乡

> 英文代号 **VoxelCraft**——仓库名、包名、构建产物(如 `VoxelCraft.apk`)与日志前缀 `[vc]` 沿用此代号;
> 游戏内一切玩家可见文案只使用《永远的山根乡》。

在浏览器里运行的跨平台(电脑 / 安卓 / iPhone)体素沙盒联机游戏。
无构建步骤、无外部资源依赖:克隆下来 `npm start` 即玩,局域网内手机无需外网。
核心理念:**谁创建房间,谁的浏览器就是这个房间的服务器**——后端只是一个不懂游戏的"哑中继"。

## 快速开始

```bash
npm install
npm start
```

需要 Node.js(项目按 Node 24 开发)。Windows 下也可以直接双击 `start.cmd`,自动开服并打开游戏页面。
只想跑服务器、不想装依赖?可以用压缩编译版:`node dist/server.js`(见下文"仓库结构")。

启动后控制台会列出所有可访问地址:

- 电脑本机:`http://localhost:8080`
- 手机(连**同一个 Wi-Fi**):访问控制台里列出的局域网地址,例如 `http://192.168.1.5:8080`

> 默认端口 8080,用环境变量 `PORT` 修改:PowerShell 下 `$env:PORT=8081; npm start`,
> macOS / Linux 下 `PORT=8081 npm start`。
> Windows 第一次启动时防火墙若弹窗询问是否允许 Node.js 联网,
> 必须勾选"专用网络"并允许,否则手机连不上。

## 玩法

打开游戏即是**世界入口首页**:低清世界渲染做背景,正中显示当前人物、当前房间
和房间状态(「在线 n 人」/「不在线·有存档」/「不在线」/「未连接服务器」),
下面三个按钮——「进入世界」「人物设定」「房间设定」。

1. **「进入世界」一键开玩,全程零确认**:
   - 还没有人物?自动随机生成一个(随机名字 + 随机外观)并保存为当前人物;
   - 有上次进过的房间就直接回去(在线就加入);
   - 房间不在线但本地有存档 → 用存档(种子 + 全部修改记录)**原样重建世界**;
   - 什么都没有 → 自动起一个随机中文房名(如"迷雾森林42")直接开公开新房。
2. **人物设定**(首页 → 人物设定):先看到当前人物的展示页(放大预览 + 名字);
   「更换人物」进入人物列表(每行可「使用」「编辑」,也可「建立新人物」或「随机建立」);
   编辑器里起名字(1–16 字)、选 **8 种素体**(男 / 女 × 高 / 矮 × 瘦 / 壮)、
   **上衣 8 色、裤子 8 色、肤色 6 色**三组配色,实时预览,「保存并使用」即生效,
   也可「随机设计」一键随机。外观所有玩家可见;人物只存在浏览器本地(localStorage),
   无需注册任何账号。目前只有「人类」这一个大类,机器人等六大类为**规划中**。
3. **房间设定**(首页 → 房间设定):先看到当前房间的展示页(名字 + 状态,可直接「进入房间」);
   「更换房间」进入房间选择页:
   - 顶部是服务器地址小节(默认折叠成一行,点开修改,规则见下节);
   - 输入房间名(支持中文等任意 Unicode,1–16 个字符)点「查找/进入」——
     在线就直接进;不在线但本地有存档就用存档重建;都没有则提示
     "没有找到「××」",确认后一键建房。**留空直接点 = 随机开新房**;
   - 三段房间列表,从上到下:**当前房间** → **历史房间**(本地最多记 20 条,
     每行「使用」「移除」;「移除」会连同对应的本地世界存档一起删)→
     **搜索到的公开房间**(最新在前、最多 50 个,每行「使用」「隐藏」;
     已在历史里的和被隐藏过的不再重复显示),点「刷新」更新;
   - 「建立新房间」= 明确开一个**全新世界**(不查存档),可勾选公开 / 私密
     (私密房不出现在列表里,只能凭房间名进入)。
   - **★ 金色房名 = 这个房间是我创建过的**(只表示"创建过",不表示"现在还是房主")。
4. **世界不消失(本地存档)**:房主**和成员**的浏览器都会自动保存世界——
   进房即存,之后每次方块改动后约 2 秒内补存,退出时立即存。所有人离开后房间
   会从服务器上消失,但**任何进过这个房间的人**之后都能凭本地存档一键原样重建它。
   每个浏览器最多保存 8 个世界(满了淘汰最久未玩的),单个世界最多 6 万条方块修改;
   存档只在 localStorage 里,**清浏览器数据就会丢档**,也不跨设备同步。
5. **房主退出,世界照转**:房主离开(主动退出或断线)时,中继自动把最早加入的
   成员提升为新房主,世界无缝继续;最后一个人也离开时房间才从服务器消失
   (此后靠上一条的本地存档重生)。
6. 同一房间(上限 8 人 = 房主 + 7 名成员)内:实时看到彼此(含素体与三组配色)、
   共享地形、方块修改全员同步,后加入的人也能看到之前的所有改动。
7. 分享房间:把 `服务器地址/?room=房间名` 链接发给朋友,打开后房间名已自动填好。
8. 游戏内**右上角「退出」按钮**随时回到入口首页(左上角显示当前房间名和人数)。

### 服务器地址栏

房间选择页顶部的"服务器地址"小节(默认折叠)决定连接哪个中继服务器:

- **留空 = 本站**:连回提供这个页面的服务器(网页版通常就这样用)
- `主机:端口`(如 `192.168.1.5:8080`)→ 按普通 `ws://` 连接(局域网风格)
- 不带端口的裸域名(如 `xxx.trycloudflare.com`)→ 按加密 `wss://` 连接(隧道/云风格)
- 写明 `http://`、`https://`、`ws://`、`wss://` 前缀时按前缀来

填过的地址会被记住,下次打开自动填上。

## 安卓 App 与公网入口

- 打包 APK:`powershell -File tools/build-apk.ps1 -DefaultServer "你的公网地址"`
  → 产物 `public/VoxelCraft.apk`(约 4.5MB,手机访问 `服务器地址/VoxelCraft.apk`
  即可下载安装;构建需要 JDK 与 Android SDK 环境)
- App 菜单里的"服务器地址"随时可改,打包时烤入的只是默认值,
  所以**换服务器/换隧道不用重装 App**
- 临时公网入口(测试用):`cloudflared tunnel --url http://localhost:8080`
  → 得到一个 `https://xxx.trycloudflare.com` 临时域名(每次重启域名会变)。
  长期部署方案见 `docs/scenarios/` 中的部署场景。

| 位置 | 内容 |
|---|---|
| `docs/whitepaper.md` | 白皮书:项目理念与整体设计的完整阐述 |

## 技术结构:人人可开服

- **谁点"创建房间",谁的浏览器就是这个房间的服务器**——权威房间逻辑在
  `public/js/host.js`,纯事件驱动(不依赖定时器,后台标签页也能正常当房主)
- `server/index.js` 只是"哑中继":房间目录 + 不透明转发,**完全不解析游戏数据**。
  把它部署到任何一台有公网地址的便宜服务器上,所有人就都能远程开房/进房
- **房主迁移**:房主掉线时中继提升最早加入的成员接任,新房主向全房间广播
  完整修改集(resync),世界无缝继续——能这么做是因为每个玩家本地都持有
  完整世界(种子 + 全部修改记录)
- **世界本地存档**:同一份"种子 + 修改集"也会落到每个玩家(房主与成员都算)的
  localStorage 里,房间从服务器消失后,任何进过房的人都能凭它原样重建世界
- 世界由种子在各端本地生成,网络上只传"方块修改"和"玩家位置"(约 12 Hz),流量极小
- 调试:浏览器控制台 `window.__vc`(world / player / hostRoom / mode / doPlace 等)
- 测试工具:`node tools/relay-test.js`(中继协议自测,自带独立端口的临时服务器,
  逐项 PASS/FAIL);`tools/host-bot.js` / `tools/member-bot.js`(模拟房主/成员)

## 当前限制

- 存档只在各玩家浏览器本地:上限 8 个世界、单世界 6 万条方块修改,
  **清浏览器数据即丢档**,不跨设备同步;服务器侧没有任何持久化
- 私密房间只靠房间名挡人,公网长期部署建议自行加访问控制
- 手机当房主可行(游戏会尽量保持屏幕常亮),但锁屏会冻结房间——
  好在会自动迁移给其他在线成员;长期房主建议用电脑
- 无生物、无合成、无水;方块共 8 种——内容层留给后续重新设计;
  人物暂只有"人类"一个大类(机器人等六大类规划中),配件系统未开放
