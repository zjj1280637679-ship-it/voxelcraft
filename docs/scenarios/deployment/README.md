# 部署与使用场景集

本目录收录 VoxelCraft 从"本机开黑"到"自建公网中继"的典型部署与使用场景。
每篇按统一结构编写:**目标 → 前提 → 操作步骤 → 预期结果 → 常见故障与排查**,
彼此独立,可按需单篇阅读。

架构背景一句话:`server/index.js` 只是一个游戏无关的"哑中继"(房间目录 + 不透明转发),
权威房间逻辑跑在房主玩家的浏览器标签页里(`public/js/host.js`)。
所以"部署"在本项目里只意味着两件事:**把中继跑起来**,以及**让玩家的设备能连到它**。
完整架构契约见仓库根目录的 `DESIGN.md`。

## 通用事实速查(均以代码为准)

- 中继默认监听 `0.0.0.0:8080`,端口由环境变量 `PORT` 覆盖(`server/index.js`)。
- WebSocket 路径固定为 `/ws`,JSON 文本帧,单帧上限 64 KB。
- 每房间上限 **8 人**(房主 + 7 名成员),超出报 `full`。
- 房间名即身份:全 Unicode,服务器做 NFC + 去首尾空白 + 合并连续空白的规范化,
  查找键为规范化后小写;规范化后须为 1–16 个码点,否则报 `bad-name`。
- 服务器地址栏规则(`public/js/network.js` 的 `relayUrl`):留空 = 本站;
  裸 `host:port` 按 `ws://`(局域网风格);裸域名(无端口)按 `wss://`(隧道/云风格);
  显式写 `http(s)://`、`ws(s)://` 前缀则以前缀为准;地址中的路径/查询串会被剔除。
