# 场景 03:手机流量进房(安卓 APK)

## 目标

在安卓手机上安装 VoxelCraft App,通过移动网络(或任意 Wi-Fi)连接远端服务器进房——
不依赖"和服务器同一局域网",也不用每次在浏览器里敲地址。

iPhone 用户:本项目没有 iOS 安装包,直接用 Safari 打开服务器网页地址即可,
游戏功能与 App 一致(App 本质是同一套网页的 Capacitor 壳)。

## 前提

- 拿到一个 `VoxelCraft.apk`。注意:**仓库不直接附带 APK**
  (`public/VoxelCraft.apk` 在 `.gitignore` 中),来源有二:
  - 服主已构建并开着服务器:手机浏览器访问 `服务器地址/VoxelCraft.apk` 直接下载
    (服务器对 `.apk` 返回正确的安装包 MIME 类型);
  - 自己构建:见[场景 05:构建安卓包](05-build-apk.md)。
- 一个可达的服务器地址:局域网 IP(场景 01)、隧道域名(场景 02)
  或自建中继(场景 06)。

## 操作步骤

1. **下载并安装 APK**。当前构建脚本产出的是 debug 签名包,
   安装时安卓会要求允许"安装未知来源应用"——在系统弹窗里对浏览器
   (或文件管理器)授予该权限后继续安装。应用 ID 为 `com.voxelcraft.app`,
   名称 VoxelCraft(见 `capacitor.config.json`)。
2. **打开 App**。首次进入先建人物(名字 + 上衣/裤子颜色)。
3. **确认服务器地址**。大厅页顶部有"服务器地址(留空 = 本站)"输入框:
   - 如果打包时用 `-DefaultServer` 烤入了默认服务器(机制:构建脚本把
     `window.VC_DEFAULT_SERVER = '...'` 写进 App 内置的 `public/app-config.js`
     副本),输入框会预填该地址,通常不用动;
   - 没有默认值或要换服务器,手动填一个,**规则**(`public/js/network.js` 的
     `relayUrl`,实际连接为该地址 + 固定路径 `/ws`):

     | 填法 | 实际连接 | 适用 |
     |---|---|---|
     | `192.168.1.5:8080`(裸 host:port) | `ws://192.168.1.5:8080/ws` 明文 | 局域网 / 裸 IP 自建服 |
     | `xxx.trycloudflare.com`(裸域名无端口) | `wss://xxx.trycloudflare.com/ws` 加密 | 隧道 / 带 TLS 的云服务 |
     | `https://...` 或 `wss://...` 前缀 | 强制 `wss://` | 显式指定加密 |
     | `http://...` 或 `ws://...` 前缀 | 强制 `ws://` | 显式指定明文 |
     | 留空 | 连"本站" | 仅网页版有意义;App 不是由中继发出的页面,留空连不上,必须填地址 |

     地址里的路径、查询串会被剔除,只留主机(和端口)。
   - 输入的值持久保存(localStorage 键 `vc-server`),且**优先级高于烤入的默认值**
     ——所以换服务器/换隧道域名只需改这一栏,**不用重装 App**。
4. **进房**:点"刷新"拉取公开房间列表,点房间行进入;
   或直接输入房间名"进入房间"(没有则提示创建,自己当房主)。

## 预期结果

- 手机走流量即可与异地玩家同房间互见、共同建造。
- App 与网页玩家完全互通(同一协议、同一服务器)。
- 注意:手机当房主可行,但**锁屏会冻结房主标签页**,房间会自动迁移给其他成员
  (托管时 App 会尽量申请屏幕常亮 wake lock);长期当房主建议用电脑。

## 常见故障与排查

| 现象 | 原因与处理 |
|---|---|
| 安装被拦截"禁止安装未知应用" | 系统设置 → 应用 → 特殊应用权限 → 安装未知应用,给下载它的浏览器授权;debug 签名包属正常现象。 |
| 大厅显示"未连接(检查服务器地址后点刷新)" | ① 地址写错或服务器没开;② 该填 `host:port` 的写成了裸域名(或反之)——按上表核对明文/加密;③ 手机所在网络封了非常规端口,试试场景 02 的隧道(443 端口几乎不会被封)。 |
| 换了隧道域名后连不上 | 隧道重启域名就变(场景 02),在服务器地址栏填新域名点刷新即可;旧值会被覆盖保存。 |
| 填了地址但每次打开 App 又变回去 | 输入框初始值优先级为:上次保存值 > 烤入默认值 > 空。正常情况下保存值生效;若清除了 App 数据,会回落到烤入默认值。 |
| 房间列表为空但朋友说房间开着 | 私密房不进列表,直接输入房间名进入;另外确认两台设备连接状态行显示的是同一个服务器。 |
| 进房名字报错 | 房间名规范化后须为 1–16 个码点(`bad-name`);房满 8 人报 `full`;房间不存在提示创建。 |
| 想用旧版 App 连新服务器 | 可以。App 壳只是装了网页和默认地址,协议在服务器端无版本协商;改服务器地址即可。但游戏页面本体由 App 内置,若仓库代码大版本升级导致协议不兼容,则需要重新打包安装(见场景 05)。 |
