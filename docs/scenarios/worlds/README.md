# 示例世界（场景）集

这里的每个 JSON 文件是一个可以"搬进"房间的小建筑场景：小屋、瞭望塔、像素画、木桥。
配套加载工具是 `tools/load-scene.js` —— 它以普通成员身份加入一个已存在的房间，把场景里的方块逐个发给房主，等全部回声确认后退出。

VoxelCraft 的世界 = **种子（程序化地形）+ 修改集（玩家放置/拆除的方块差量）**。场景文件本质上就是一份预制的修改集。

## 场景列表

| 文件 | 名称 | 绑定种子 | 位置（出生点在 (8, 8) 附近） | 方块数 |
|---|---|---|---|---|
| `cabin.json` | 林间小屋 | 37 | x −17..−13，z 15..19，地面 y=23 | 104 |
| `watchtower.json` | 瞭望塔 | 15 | x 5..10，z 5..10，平台 y=38（骑跨出生点，进房即出生在塔顶） | 94 |
| `pixel-heart.json` | 像素爱心 | 25 | z=8 竖面，x 2..12，y 32..43 | 72 |
| `bridge.json` | 跨谷木桥 | 7 | x 11..32，z 6..8，桥面 y=27，谷深 8 格 | 85 |

## 文件格式

```json
{
  "name": "林间小屋",
  "seed": 37,
  "description": "给人看的简介（加载器忽略）",
  "blocks": [ [-17, 23, 15, 7], [-17, 23, 16, 7], ... ]
}
```

- `seed`（整数）、`blocks`（数组）为必填（`tools/load-scene.js` 加载时校验）；`name`（字符串）建议提供——加载器不强制，缺失时在种子不匹配等提示里用文件名代替显示；`description` 可选，仅供阅读。
- `blocks` 的每个元素是 `[x, y, z, id]`，四个都必须是**整数**。
- 取值范围与房主校验规则一致（见 `public/js/host.js` 的 `block` 分支）：`0 <= y < 64`（世界高度 `HEIGHT`，见 `public/js/constants.js`），`0 <= id <= 8`。越界条目会被房主静默丢弃，所以 `tools/load-scene.js` 在发送前先做同样的校验并直接报错。
- 方块 id（来自 `public/js/constants.js` 的 `BLOCK`）：

  | id | 方块 | id | 方块 |
  |---|---|---|---|
  | 0 | AIR（拆除） | 5 | LOG 原木 |
  | 1 | GRASS 草方块 | 6 | LEAVES 树叶 |
  | 2 | DIRT 泥土 | 7 | PLANK 木板 |
  | 3 | STONE 石头 | 8 | BRICK 砖块 |

## 为什么场景要绑定种子

地形完全由种子决定：`public/js/terrain.js` 的 `heightAt(x, z) = round(26 + n1(x·0.012, z·0.012)·10 + n2(x·0.05, z·0.05)·4)`，并截断到 [4, 50]（`n1`、`n2` 是以 `seed` 和 `seed+1337` 播种的两套 simplex 噪声；`h <= 21` 的柱子顶面是沙滩）。

场景里的坐标，是在 Node 中直接 `import` 上述地形代码、按**该种子的真实地表高度**逐柱算出来的：小屋的地板贴着 y=23 的平地，塔腿和桥墩各自落到所在柱子的地面上。换一个种子，同样的坐标下地表高度就变了——建筑会悬空在半空或埋进山体里。因此 `tools/load-scene.js` 会比对房间的种子（`joined` 消息携带）与场景的 `seed`，不一致时拒绝加载（可用 `--force` 强行覆盖，后果自负）。

## 如何用指定种子开一个房

**浏览器 UI 不支持指定种子。** 浏览器里点"创建这个房间"时，种子在 `public/js/main.js` 的 `onHosted` 回调中随机生成（`Math.floor(Math.random() * 2 ** 31)`）。要得到固定种子的房间，用测试房主机器人：

```
node tools/host-bot.js <房间名> [relayHost[:port]] [seed]
```

例如 `node tools/host-bot.js 我的小屋 localhost:8080 37`（seed 缺省为 12345，relay 缺省为 `localhost:8080`）。host-bot 创建的房间默认公开，会出现在浏览器大厅列表里；它在 `joined` 消息里下发该种子，浏览器成员据此生成一模一样的地形。

想要"真人房主 + 指定种子"：先用 host-bot 开房，让一个浏览器玩家加入，然后 Ctrl+C 退出 host-bot——中继会把最老的成员提升为新房主，世界（种子 + 全部修改）原样保留。注意：若 host-bot 退出时房里没有任何成员，房间即被删除，已加载的场景也随之消失。

## 加载步骤

1. 启动服务器（已有在跑的可跳过）：`npm start`，或换端口 `PORT=18302 node server/index.js`。
2. 用场景绑定的种子开房：`node tools/host-bot.js 我的小屋 localhost:8080 37`。
3. 加载场景：

   ```
   node tools/load-scene.js docs/scenarios/worlds/cabin.json 我的小屋 localhost:8080
   ```

   可选参数：`--interval=<ms>` 调整逐块发送间隔（默认 40ms，最小 10ms）；`--force` 忽略种子不匹配。
4. 浏览器打开服务器地址，进入同名房间参观。先进后进都可以：晚加入的成员会通过 `joined` 消息的存量修改集拿到全部方块。

加载器的退出条件：房主对每条 `block` 的**回声**（协议规定回声发给包括发起者在内的所有成员）全部到齐后打印统计并以 0 退出；连续 15 秒没有新回声则报告缺失并以 1 退出。若加载途中发生房主迁移，新房主广播的 `resync` 也会被用来补记确认。

## 注意事项

- **限速**：每条 `block` 消息本身只有几十字节，远小于中继 64KB 的单帧上限；逐块限速主要是为了不刷屏、给房主标签页留出处理时间。真正接近 64KB 上限的是 `joined`/`resync` 里的整份修改集（数千条的量级）——示例场景都在 ~100 块，叠加余量充足，但请避免往同一房间塞入几十个大场景。
- **往热闹的房间加载**：如果别的玩家恰好同时编辑同一格，房主回声的 id 可能与场景不符，该条目会确认不上（最终以超时报缺失的形式呈现）。建议往安静的房间加载。
- **自制场景**：在 Node 里 `import { makeGenerator } from '../public/js/terrain.js'`（以仓库根为工作目录时路径自行调整），用 `makeGenerator(seed).heightAt(x, z)` 取地表高度再摆方块。还要避开树：顶面为草（`h > 21`）且 `hash2(x, z, seed) < 0.01`（`public/js/noise.js`）的柱子上有一棵树冠半径 2 的树，设计时让建筑范围外扩 2 格内都没有树即可。
